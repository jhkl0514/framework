Design Code

Modal

• 모달은 사용자에게 특정 작업에 대해 알리고 사용자가 결정을 내려야 하는 중요한 정보를 포함하여 제공한다.

구조

  • 1오버레이 :

    모달과 하단의 기본 창을 시각적으로 구분하기 위한 배경

  • 2타이틀 :

    모달의 상단 영역으로 모달에서 제공하는 콘텐츠나 요청하는 행동에 대한 간단한 요약 정보를 담은 제목, 설명을 제공함

  • 3닫기 :

    사용자가 모달을 닫을 수 있게 하는 버튼 요소로 대개 "X" 아이콘으로 표현됨.
    모달에서 데이터를 입력하여 제출하거나 행동을 확정해야 하는 경우, 닫기 버튼은 모달을 닫는 동시에 취소와 동일한 기능을 제공함

  • 4본문 :

    텍스트, 이미지 또는 다른 컴포넌트 요소가 제공되는 영역

  • 5버튼 :

    모달의 하단 영역으로 취소, 확인, 삭제, 저장 등 과업을 완료하거나 중단하기 위한 주요 액션 버튼을 제공함.
    본문에 입력폼을 제공하는 경우, 입력폼과 관련된 컨트롤은 본문 영역에 배치할 수 있으나 행동을 확정하거나 취소하는 액션은 항상 하단에 제공해야 함

타입

기본팝업

• PC,모바일 공통적으로 사용하는 기본형태

바텀시트

• 모바일에서만 사용하는 형태

지도팝업

• PC 화면 중 지도에서 노출되는 형태

공지팝업- 이미지형

• 팝업 사이즈 : 480x600(고정), 이미지형 팝업은 여러개 등록 가능하도록 이미지 롤링 처리한다.

• 이미지 사이즈 : 440x400(고정), 이미지 사이즈 내에서 자유롭게 배너 이미지 제작한다.

• 버튼은 1개 또는 2개를 사용할 수 있으며, min-width : 150px , padding : 0 16px 값을 같도록 설정

공지팝업-이미지+텍스트형

• 팝업 사이즈 : 사이즈 : 480x600(고정), 타이틀 : 한줄, 글자수 40자, 상세내용 : 최대 4줄, 글자수 138자. 상세 내용 높이값을 고정으로 처리한다.

• 이미지 사이즈 : 440x230(고정), 이미지 사이즈 내에서 자유롭게 배너 이미지 제작한다.

• 버튼은 1개 또는 2개를 사용할 수 있으며, min-width : 150px , padding : 0 16px 값을 같도록 설정

사용법

  • • 모달은 콘텐츠의 양을 고려하여 적절한 크기로 제공한다.

  • 모달의 콘텐츠가 짧은 메시지인 경우에는 모달의 너비를 최소화하여 문장 길이(line length)가 길어지는 것을 피해야 하고 콘텐츠가 상대적으로 복잡한 경우에는 전체 화면 너비를 차지하는 모달의 사용을 고려해야 한다.

  • 모달의 콘텐츠가 짧은 메시지인 경우에는 모달의 너비를 최소화하여 문장 길이(line length)가 길어지는 것을 피해야 하고 콘텐츠가 상대적으로 복잡한 경우에는 전체 화면 너비를 차지하는 모달의 사용을 고려해야 한다.

  • • 콘텐츠 영역에 스크롤이 필요한 경우 사용자가 스크롤 영역을 인지할 수 있도록 표현하고 하단 버튼 영역이 항상 표시되도록 한다.

  • 모달 콘텐츠는 간결하고 모달에서의 상호작용은 최소로 제공해야 한다. 그러나 사용자별 디바이스 화면 크기 차이, 모달 유형 등에 따라 불가피하게 콘텐츠 영역에 세로 스크롤이 생성되는 경우 스크롤바를 제공하여 사용자가 스크롤 해야 함을 인지할 수 있도록 해야 한다. 또한 액션 버튼이 스크롤 한 영역의 가장 아래에 배치될 경우 사용자에게 혼동을 줄 수 있으므로 액션 버튼이 제공되는 푸터는 모달 영역의 하단부에 항상 표시될 수 있도록 구현해야 한다.

  • • 모달 내 상호작용을 최소화한다.

  • 모달 내부에 액션 버튼 외에 상호작용이 필요한 다른 컴포넌트를 사용하는 것을 피해야 한다. 탭, 아코디언, 입력폼 같은 컴포넌트가 모달 내부에 제공될 경우 사용자는 모달과 상호작용 중인 맥락을 잊어버리기 쉽고, 사용자의 이용 환경에 따라 화면 크기가 줄어드는 경우 상호작용이 어렵다.
