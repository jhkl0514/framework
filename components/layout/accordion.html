Design Code

Accordion

• 아코디언은 섹션의 중요한 세부 사항을 먼저 강조 표시하고 확장되면 더 많은 세부 정보를 표시하여 점진적으로 정보를 공개한다. 컨텐츠 전체를 읽는 것이 중요하지 않은 경우 아코디언을 사용하여 페이지 스크롤을 줄일 수 있다.

구조

  • 1패널 :

    콘텐츠를 감싸고 있는 영역

  • 2타이틀 :

    콘텐츠의 주요 내용이나 제목

  • 3펼침/접힘 버튼:

    펼침/접힘 상태를 나타냄

  • 4내용:

    텍스트 또는 컴포넌트를 조합하여 표현

스타일

텍스트형

• 타이틀 영역과 내용 영역이 하나의 콘텐츠 섹션으로 인지될 수 있게 표현한 형태이다.

• 내용영역에는 주로 텍스트만 사용한다.

• 연속해서 아코디언을 배치할 경우 하단 라인(#E0E0E0)을 사용할 수 있다.(선택)

박스형(Outlined)

• 타이틀 영역과 내용 영역을 분리시킨 형태로 내용이 많은 경우 주로 사용한다.

• 내용 영역에는 텍스트 및 컴포넌트들을 조합하여 사용할 수 있다.

• 필요에 따라 패널 간의 여백을 조정할 수 있으며, 그림자를 사용할 수 있다.

타입

텍스트형
컴포넌트 조합형

• 텍스트와 컴포넌트를 조합해 사용하는 유형이다.

사이즈 및 간격

기본형
박스형

사용법

  • • 접힘 상태를 기본으로 한다.

  • 단, 외부 링크에서 랜딩하여 특정 아코디언 패널로 진입한 경우, 내용이 꼭 노출되어야 할 경우 펼침 상태를 기본으로 한다.

  • • 여러 개의 아코디언 패널이 사용될 경우, 패널을 1개씩 펼치고 접을 수 있다.

  • 펼친 패널은 접기 전까지 펼쳐진 상태를 유지하고 있으나 다른 패널을 펼치면 현재 펼쳐진 패널은 자동으로 접히게 된다.
    단, 패널간의 내용의 비교가 필요한 경우는 수동으로 접히게 한다.

  • • 타이틀은 최대 1줄까지 기재할 것을 권장한다.

  • 타이틀 외 텍스트는 패널 영역이 펼쳐졌을 때 노출한다. 패널이 접힌 상태에서는 버튼을 배치하지 않는다.
