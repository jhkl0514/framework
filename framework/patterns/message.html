Message

사용자에게 정보를 전달하거나 상호작용을 유도하기 위해 사용되는 기능이다. 알림에는 Modal, Toast, Tooltip이 포함되며 각각의 특징과 사용 목적이 다르다.

유형

  • • Modal:

    화면 중앙에 배치되며 중요한 정보를 사용자에게 전달하거나 사용자로부터 중요한 입력을 받을 때 사용한다.

  • • Toast:

    사용자에게 짧은 시간 동안 간단한 알림이나 피드백을 제공할 때 사용한다. 성공/실패, 새로운 메시지 도착 등의 알림을 표시할 때 사용한다.

  • • Tooltip:

    특정 요소에 대해 추가적인 정보를 제공할 때 사용한다. 버튼, 아이콘 등의 사용법을 설명하거나 추가 정보를 제공할 때 사용된다.

  • • System Alert:

    현재 정해진 규칙은 없음.(※ 기획-개발자와 논의 후 업데이트 예정)

특징 모달 (Modal) 토스트 (Toast) 툴팁 (Tooltip)
오버레이 있음 없음 없음
표시시간 사용자가 직접 닫을 때까지 계속 표시됨 짧은 시간(2초) 동안 표시되고 자동으로 사라짐 마우스 클릭 시 표시되고 닫기 버튼 또는 일정시간이 지나면 사라짐
간섭정도 사용자 집중을 요구하며 다른 작업 차단 비간섭적 비간섭적
위치 화면 중앙 화면 하단 특정 요소 위,아래, 옆
사용사례 상세 정보 제공, 중요한 확인 작업 확인, 에러 메시지, 간단한 알림 버튼, 아이콘, 기능 등의 간단한 설명

구조

a. 모달_메시지형

• MO: SM-340사이즈, PC: MD-480사이즈를 사용한다.

• 모바일에서는 닫기 버튼을 노출하지 않고 하단 버튼(최대 2개)만 사용한다. 또한 본문 내용이 없을 경우 타이틀만 노출하고 그 외 구성요소(구분선/본문내용)는 숨김처리한다.

• PC에서는 닫기 버튼은 고정값으로 노출하며, 하단버튼(최대 2개)을 사용한다. 또한 타이틀, 본문 내용은 필수값으로 들어가야 한다.

  • 1모바일메세지 팝업
  • 2PC 메세지 팝업
b. 토스트_메시지형

• MO: 화면의 80%, PC: 520px 고정값을 사용한다.

• 토스트 메세지는 노출 2초 후 사라지도록 설정한다.(닫기 버튼이 없을 경우)

• 자세한 상세정보는 컴포넌트_토스트 참고

  • 1모바일메세지 토스트
  • 2PC 메세지 토스트
c. 툴팁_메시지형

• MO/ PC 컴포넌트는 공통으로 사용한다.

• PC : 최소 150px, 최대 500px (모바일에서 최대 크기는 논의 후 업데이트 예정)

• 자세한 상세정보는 컴포넌트_툴팁 참고
