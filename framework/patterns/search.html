Search

검색은 사용자가 큰 데이터 집합에서 원하는 정보를 찾을 수 있도록 도와주는 기능이다.
탐색 수단을 통해 원하는 콘텐츠를 찾지 못하는 상황에는 특정 정보와 관련된 단서를 제공함으로써 사용자가 필요한 콘텐츠를 쉽게 찾을 수 있도록 해준다.

유형

  • • 통합검색:

    검색을 통해 서비스의 데이터 구조와 활용할 수 있는 정보를 더 쉽게 파악할 수 있으며, 일반적으로 헤더 영역에 배치되며, 검색 결과는 별도의 결과 화면에서 제공된다.

  • • 부분검색:

    단일 화면 서비스, 특정 정보 목록, 표와 같은 소규모 데이터 집합을 검색하는 데 사용하며, 데이터 집합 상단에 배치되며, 검색 결과는 데이터 집합에 직접 반영된다.

검색 기능

  • • 검색어 입력:

    검색어를 입력하는 과정에서 참고할 수 있는 정보를 제공받고 활용하는 과정 이다.

  • • 상세 검색:

    상세 검색, 정렬/필터링 수단에 대한 탐색 수행하며, 검색 결과 목록을 원하는 범위로 좁히기 위해 상세 조건을 설정 후 적용 결과를 확인 한다.

  • • 현재 신규 UI에서는 “상세 검색" 기능만 제공하고 있다. (※ 신규 UI 추가 시 업데이트 예정)

구조 (상세검색)

a. 필터 + 검색어 검색

• 기본 정렬 및 필터 컨트롤을 사용하고 검색과 검색어 입력하여, 세부 필터 유형과 수에 따라 목록 상단에 배치되는 패널형태로 제공한다.

• 검색 조건이 많을 경우, 첫 줄에 필드값과 상세 버튼을 노출한다.

  • 1접기 (Default):

    상세검색 기본은 “접기" 형태이며, [상세] 버튼을 누르면 ‘상세'화면이 펼쳐진다.

  • 2펼치기 (Detail):

    필터 컨트롤 및 검색어 입력 후 [검색] 버튼을 누르 ‘검색 결과 목록’이 표시된다.

  • 2a선 (line) 표시 방법:

    패널 영역 첫 번째 상단의 하단 라인은 ‘접기 상태 > 선 숨기기’, ‘펼침 상태 > 선 표시’ 로 제공한다.

  • 2b검색 버튼:

    [초기화], [검색] 버튼 순으로 배치하며, [초기화] 버튼은 ‘line' 형태, [검색] 버튼은 ‘Filled’ 형태로 제공한다.

b. 드롭다운 검색 (* 체크 요소 확인 필요!)

• 콘텐츠 영역이 좁을 경우, ‘검색' 타이틀과 ‘드롭다운' 아이콘을 노출한다.

  • 1드롭다운 표시:

    콘텐츠 영역이 좁을 경우, ‘검색' 타이틀과 ‘드롭다운' 아이콘을 노출한다.

  • 2혼합 검색:

    타이틀, 열, 버튼 등 다른 요소들과 조금씩 기준이 다름 (이유 : 콘텐츠 영역이 좁아 별외 내용이 적용 됨)

c. 필터 + 태그칩 검색

• 사용자가 검색 옵션을 간편하게 선택하고 원하는 정보를 빠르게 찾을 수 있도록 돕는 강력한 형태입니다.

  • 1필터 검색:

    셀렉트 필드를 직관적으로 선택 후 [검색] 버튼을 누르면 검색 데이터가 표시된다.

  • 2태그칩 검색:

    검색 결과와 관련된 키워드나 태그를 표시하는 칩으로 사용자가 이 칩을 클릭하여 해당 키워드나 태그로 검색 결과를 좁힐 수 있다.

d. 태그칩 검색

• 사용자가 검색 옵션을 간편하게 선택하고 원하는 정보를 빠르게 찾을 수 있도록 돕는 강력한 형태입니다.

사이즈 및 간격 (상세검색)

  • 1고정값 요소 :

    타이틀 120px, 셀렉트 필트 160px

  • 2외부 패딩 값:

    전체 패널 16px

  • 3내부 패딩 값:

    li 요소 : 12px, li Gap: 12px, 셀렉트 + 인풋 Gap 12px, 버튼 Gap: 8px

  • 4버튼 사이즈:

    전체 검색 : 130 x 36 (px), 필드 검색 : 상대값

  • 1고정값 요소 :

    타이틀 100px, 셀렉트 필트 100px

  • 2외부 패딩 값:

    상단 검색 전체 패널 16px, 하단 검색 조건 패널 8px, 16px, 12px, 16px

  • 3내부 패딩 값:

    li요소 위/아래 8px, 선택필드 Gap 16px, 셀렉트 + 인풋 Gap 12px,

  • 4버튼 사이즈:

    전체 검색 : 108 x 36 (px)

Search

검색은 사용자가 큰 데이터 집합에서 원하는 정보를 찾을 수 있도록 도와주는 기능이다.
탐색 수단을 통해 원하는 콘텐츠를 찾지 못하는 상황에는 특정 정보와 관련된 단서를 제공함으로써 사용자가 필요한 콘텐츠를 쉽게 찾을 수 있도록 해준다.

구조 (상세검색)

a. 검색 필터형(다중)

• 검색 필터가 2개 이상일 경우 해당 영역에 노출하여 검색하는 방식

• 선택된 조건값은 하단 검색값 노출 영역에 표시한다. 필터 내 모든 조건을 초기화 할 수 있는 초기화 버튼을 제공한다.

  • 1카테고리 필터:

    검색에 필요한 카테고리 필터들이 노출된다.

  • 2검색 버튼:

    검색 가능한 모든 필터 항목이 노출된다.

  • 3선택된 필터 값 노출 :

    내가 선택한 조건값이 노출된다. 단일 삭제도 가능하다.

  • 4초기화 버튼:

    모든 조건값을 초기화 할 수 있다.

b. 단일 검색형

• 검색 필터가 1개인 경우 카운트 영역 오른쪽에 셀렉트 영역을 터치하여 검색하는 방식

• 선택된 조건값은 셀렉트박스 텍스트 영역에 노출하여 보여준다.

  • 1검색 건수 노출 :

    조건값에 따라 검색된 결과값을 노출한다.

  • 2검색필터 영역:

    셀렉박스를 터치하여 검색 조건값을 설정할 수 있다.

사이즈 및 간격

사용법

  • • 필터 선택 시, 정보 선택가능한 바텀시트를 노출하여 조건을 선택 및 검색 할 수 있다.

  • 각각의 필터를 선택할 때도 해당하는 내용의 바텀시트를 노출한다.

  • • 검색 필터가 1개+검색 영역이 있을 경우, 검색 필터형을 사용한다.

  • 검색 필터가 1개이지만 직접 입력 할 수 있는 검색 영역이 있을 경우는 2개의 검색 조건값으로 본다.

  • • 전체 초기화를 할 경우, 검색 조건값 영역이 비 노출된다.

  • 검색 조건값을 하나씩 삭제하여 조건값이 0개가 되었을 때, 초기화 버튼을 눌러 초기화를 했을 때는 해당 영역을 비 노출 처리한다.
