Pagination (페이지네이션)

  • Components
  • Navigation (탐색)
  • Pagination (페이지네이션)
Design Code

Pagination

• 페이지네이션은 많은 양의 콘텐츠를 탐색하기 쉽도록 여러 화면에 나누고, 분할된 화면을 탐색하는 데 사용되는 요소이다.

구조

• 분할된 화면의 순서를 숫자로 표시한다.

• 숫자 링크를 눌러 원하는 순서의 화면 목록으로 직접 이동하거나 이전/다음 버튼을 눌러 화면을 순차적으로 탐색할 수 있다.

• 첫 화면과 마지막 화면 숫자 링크가 항상 표시되므로 사용자는 목록의 시작이나 끝으로 빠르게 이동할 수 있다.

  • 1이전/다음 버튼

    화면 목록을 앞/뒤로 탐색하는 데 사용되는 버튼

  • 2숫자 링크 목록

    표시된 숫자의 화면으로 이동할 수 있는 링크

상태

  • 1Inactive

    선택되지 않은 기본상태

  • 2Hover:Pressed:Active

    호버/눌림/선택

  • 3Active

    현재 위치(선택됨)

조합

• 기본 배치 : 페이지 목록은 가운데 배치하며 단독으로 사용한다.

• 총 데이터 개수 표시 + 페이지당 데이터 개수 선택 : 페이지 목록을 가운데 기준으로 좌/우 배치한다.

  • 1페이지 목록

    선택되지 않은 기본상태

  • 2총 데이터 개수 표시

    전체 데이터 목록에 포함된 데이터 개수를 표시

  • 3페이지당 데이터 개수 선택

    사용자가 한 페이지에 표시되는 데이터 개수를 선택할 수 있도록 셀렉트 형태로 제공 (20개씩, 50개씩, 100개씩..)

사이즈 & 간격

노출 조건

• 리스트 : 10개 초과
• 테이블 : 20개 초과
• 썸네일 : 4X3 -> 12개 초과
a. 페이지 수 7개 이하
b. 페이지 수 7개 초과(8개부터)

• 최대 노출 가능한 페이지 수 이상일 경우 뒤에서 2번째 페이지 번호를 생략하고 말줄임표를 사용해 마지막 화면을 제공한다.

• 말줄임표는 페이지 번호 이동에 따라 가변적으로 적용된다
