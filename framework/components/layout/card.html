    Card (카드)

    • Components
    • Layout (레이아웃)
    • Card (카드)
    Design Code

    Card

    • 정보 제공을 목적으로 한 컨테이너이다. 다양한 정보, 기능 및 컴포넌트를 구성요소로 포함할 수 있으며, 포함된 구성요소에 따라 컨테이너의 크기가 달라질 수 있다.

    구조

    • 1 Badge(Optional) :

      사용자에게 정보를 강조하거나 특정 요소에 주의를 끌기 위해 아이콘 또는 텍스트에 추가적인 요소를 사용하여 표시한다.

    • 2텍스트 :

      카드 내 메인 텍스트가 노출되며, 상품카드의 경우 상품 타이틀명을 표시한다.

    • 3보조 텍스트(Optional) :

      보조 텍스트에는 아이템에 대한 부가적인 정보, 설명과 같은 텍스트를 표시한다.

    • 4버튼 :

      카드에는 액션버튼이 포함될 수 있다.

    • 5썸네일 :

      해당 아이템의 이미지가 들어간다.

    • 6금액 :

      해당 아이템에 대한 금액이 노출된다.

    타입

    정보 전달형

    • 하나의 주제 내 정보를 요약하거나 관련된 정보 또는 기능을 함께 제공하는 경우 사용한다.

    • 하나의 카드 내 정보와 기능은 최대 5개까지 제공할 수 있다.

    • 컴포넌트를 조합하거나 카드를 확장하여 추가 정보를 제공할 수 있다.

    아이템형

    • 사용자에게 상품 정보를 집약적으로 전달하기 위한 요소이다. 상품 정보(가격, 상품명, 혜택)를 명확하게 전달해 상세 페이지로 유도하는 목적을 가진다.

    사용법

    • • 모바일에서 정보전달형 카드형은 가로 또는 세로로 나열 할 수 있다.

    • 세로로 나열할 경우, infinite scroll 또는 더보기 버튼을 통해 카드를 추가로 더 불러올 수 있도록 한다.
      가로로 나열할 경우, 인디게이터와 함께 사용할 것을 권장합니다. 가로로 연속된 카드는 카드의 일부를 좌측 또는 우측에 노출한다.

    • • 정보전달 카드형 내 터치 영역이 많을 경우 주요 업무는 카드 자체를 눌러서 이동하며, 그 외의 기능은 버튼으로 제공한다.

    • 버튼 선택 시 카드 정보와 연관된 기능을 제공하거나 해당 정보의 상세 페이지로 이동한다.
      버튼을 포함한 터치 가능 항목은 최대 5개를 넘지 않도록 한다.

    • • 카드 내 주소 마스킹 규칙

    • 주소 문자열의 앞에서부터 공백 포함 8글자까지만 노출 -> 이후 모든 글자는 ‘*’로 치환
