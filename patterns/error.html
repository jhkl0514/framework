Error

오류는 사용자가 요청한 작업을 시스템이 완료하지 못하는 문제가 발생했을 때 제공된다.
발생 가능한 문제 를 사전에 안내하는 경고와는 달리, 오류는 이미 발생한 문제에 관한 정보를 사용자에게 알려준다.
오류정보는 단순히 사용자에게 문제가 발생했음을 알려주는 데 그치지 않고 본래 수행하고자 했던 행동을 완수할 수 있게 유도해야 한다.
오류정보를 제공하기 위해 별도의 모달을 사용하거나 오류가 발생한 지점에 직접 메시지를 전달할 수 있다.

유형

  • • 인라인 메시지:

    오류가 발생한 입력 필드나 컴포넌트 바로 옆에 표시되는 간단한 메세지를 말한다. (사용자의 시선을 문제 지점으로 집중시키고 즉각적인 피드백을 제공하는 데 효과적)

  • • 모달:

    사용자의 집중을 오류 메시지에 완전히 집중시키는 데 효과적이며, 중요한 오류나 복잡한 해결 방법을 안내할 때 유용하게 활용된다.

  • • 토스트:

    화면 하단에 짧게 나타나는 메세지로, 사용자에게 간단한 알림이나 피드백을 제공하는 데 적합하며, 화면을 가리지 않고 다른 작업을 병행할 수 있다는 장점이다.

특징 인라인 메시지 (Inline error) 모달 (Modal) 토스트 (Toast)
사용 상황 입력 오류, 필수 입력 누락, 형식 오류 등 비교적 경미한 오류에 적합 중요한 오류나 복잡한 해결 방법을 안내할 때 적합 간단한 오류, 오류 피드백 등 부가적인 정보 전달에 적합
장점 가볍고 눈에 잘 띄어 사용자의 주의를 빠르게 끌 수 있음 오류 메시지에 완전히 집중시키고, 중요한 정보를 강조 표시하는 데 효과적 화면 공간을 최소화하면서도 사용자에게 필요한 정보를 전달
단점 복잡하거나 많은 정보를 전달하기 어려움 다른 화면 요소를 가려버리고 사용자 흐름을 방해할 수 있음 사용자가 놓칠 수 있음

구조

a. 인라인 메시지

• 기본적으로 인풋, 셀렉박스의 에러표시는 에러 테두리 표시, 하단 에러메시지를 노출하여 사용

  • 1모바일메세지
  • 2PC 메세지
b. 모달

• 중요한 정보를 사용자에게 전달하거나 사용자로부터 중요한 입력을 받을 때 사용한다.

• 예시) 취소/확인이 꼭 필요한 건일 경우

c. 토스트

• 사용자에게 짧은 시간 동안 간단한 알림이나 피드백을 제공할 때 사용한다.

• 예시) 확인 또는 단순한 에러 메시지만 노출해야 되는 경우

예외 케이스 및 조합

  • • 모달, 세로형 테이블 :

    에러 테두리는 표시하지만, 하단 에러 메시지를 노출하지 않는다. 대신 에러메시지는 토스트 알럿으로 표시 한다.

a. 모달
b. 세로형 테이블
c. 가로형 테이블

Error

• 오류는 사용자가 요청한 작업을 시스템이 완료하지 못하는 문제가 발생했을 때 제공된다.
• 발생 가능한 문제 를 사전에 안내하는 경고와는 달리, 오류는 이미 발생한 문제에 관한 정보를 사용자에게 알려준다.

구조

a. Text Fields : 대대여 기준
  • • 단일 필드:

    입력 값에 오류가 발생하면 해당 필드 바로 옆에 에러 메시지를 표시.

  • • 여러 필드:

    입력 값에 오류가 발생하면 전체 입력 영역 하단에 통합된 에러 메시지를 표시.

  • • 유효성 검사:

    각 입력 영역은 유효성 검사를 거치며, 모든 오류가 해결될 때까지 다음 영역으로 이동할 수 없음

b. 모바일 카드형 UI 에러 가이드 : 대대여 기준 (출고처리)
  • • 필수 항목 미 입력 시 :

    카드 리스트의 해당 카드에 에러를 표시하고 추가적인 안내를 위한 토스트 팝업을 노출

  • • 표시 방법:

    카드 전체 또는 필수 항목이 위치한 부분을 붉은색 테두리나 배경색으로 강조하여 시각적으로 구분 (굵은 글씨체 또는 붉은색으로 강조)
