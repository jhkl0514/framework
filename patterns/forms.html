Forms

입력폼은 사용자가 데이터를 입력하여 서버로 전송하는 데 사용하는 콘텐츠 섹션으로 하나 이상의 입력 컨트롤로 구성된다.

유형

  • • 페이지에서의 데이터 입력:

    가장 기본적인 입력폼의 유형으로 1개 이상의 폼 콘텐츠로 구성된다. 입력할 데이터가 복잡할 경우 단계별로 나누어 여러페이지에 입력폼을 제공할 수 있다.

  • • 모달을 활용한 데이터 입력:

    빈번하게 입력/수정하지 않는 데이터는 모달을 활용하여 입력하며, 입력 관련 컴포넌트의 수는 10개 이내로 구성하는 것이 바람직하다.

  • • 테이블을 활용한 데이터 입력:

    테이블 내에 입력하는 컴포넌트 스타일은 자주 사용하지 않아야 하며, 스크롤되는 테이블 형태에서는 사용하지 않도록 한다.

구조

a. 기본형

• 레이아웃 구조에 따라 레이블과 인풋 컴포넌트 배치를 가로형/세로형으로 구분하여 사용할 수 있다.

• 입력폼 콘텐츠에서 필수 입력(*,필수)과 선택 입력 항목은 명확하게 구분하여 한다.

  • 1가로형 구조 입력폼
  • 2세로형 구조 입력폼
b. 확장형

• 셀렉박스 컴포넌트(세그먼트, 셀렉트, 라디오, 파일업로드 등)과 조합하여 입력하는 형태

  • 1Segmented Control (세그먼트 컨트롤) : 컴포넌트 바로가기
  • 2Select (셀렉트) : 컴포넌트 바로가기
  • 3File upload (파일 업로드) : 컴포넌트 바로가기
  • 4Textarea (텍스트 영역) : 컴포넌트 바로가기
  • 5Calendar (달력) : 컴포넌트 바로가기
  • 6Checkbox (체크박스) 컴포넌트 바로가기
C. 테이블 입력형

• 테이블 셀(td)안에 들어갈 수 있도록 입력 텍스트 수를 지정해야 한다. (두 줄 X)

• 테이블 셀(td) 안에 들어가는 텍스트를 고려하여 사이즈를 고려하여 텍스트가 잘리지 않도록 해야 한다.

• 스크롤 테이블 형태에서는 입력폼을 사용할 수 없다.
