상품주문서 페이지

• 상품주문서 페이지는 구매를 결정한 상품의 정보와 함께 배송지, 결제 정보 등을 입력하는 페이지이다.

• 주문정보 등이 명확하게 표시 되어야 하며, 배송정보 입력란은 주문 과정에서 가장 중요한 부분이므로 잘못된 정보 입력 시 오류 메시지를 표시, 다시 한번 확인하도록 기능제공한다.

구조

a. 대대여 기준 : 배송정보
  • 1수급자 목록 : Modal (모달) 사이즈 : LG-600, 스크롤 정책 적용 컴포넌트 바로가기
  • 2주소 : 설치 건 (비 활성화), 택배 건 (활성화)
  • 3필수 값 : 별표, 빨강색으로 표시
  • 4배송 입력 : 패턴 - Error(오류) - 가로형 테이블 참조 패턴 바로가기,컴포넌트 - Text input(텍스트 입력 필드) 참조 컴포넌트 바로가기
b. 대대여 기준 : 주문정보, 청구정보
  • • 주문정보:

    상품명, 상품 이미지, 옵션, 배송희망일 등 정보를 보여준다.

  • • 청구정보:

    청구 금액과 관련된 정보를 보여준다.

시스템에 따른 상태표현

a. 주소 : 설치 건

설치 건은 상품리스트, 상품상세 페이지에서 ‘지역선택'을 하고 주문 신청 페이지로 이동한다.

  • 1수급자 목록 : [수급자 목록에서 불러오기] 버튼을 눌러서 수급자 등록 하거나, 직접 등록 가능함.
  • 2지역선택 : 미리 선택한 지역 내에서만 주문 가능함. (비활성화)
  • 3주소 : 활성화/비활성화 기준

    (* 지역선택한 지역 내에서만 주소 변경 가능)

  • • 주소 입력 전 :

    입력필드 어디를 눌러도 [우편번호] 버튼과 같은 기능 함. (활성화)

  • • 우편 번호를 통한 입력 :

    입력필드 ‘우편번호 검색’, ‘주소 검색’ (비활성화), ‘상세주소 입력 (활성화)

  • • 수급자 목록을 통한 입력 :

    입력필드 ‘우편번호 검색’, ‘주소 검색’ (비활성화), ‘상세주소 입력 (활성화)

b. 주소 : 택배 건

설치 건은 상품리스트, 상품상세 페이지에서 ‘지역선택'을 하고 주문 신청 페이지로 이동한다.

  • 1수급자 목록 : [수급자 목록에서 불러오기] 버튼을 눌러서 수급자 등록 하거나, 직접 등록 가능함.
  • 2주소 : 활성화/비활성화 기준
  • 주소 입력 전 :

    입력필드 어디를 눌러도 [우편번호] 버튼과 같은 기능 함. (활성화)

  • • 우편 번호를 통한 입력 :

    입력필드 ‘우편번호 검색’, ‘주소 검색’ (비활성화), ‘상세주소 입력 (활성화)

  • • 수급자 목록을 통한 입력 :
